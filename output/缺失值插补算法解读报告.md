# 缺失值插补算法解读报告

## 一、问题背景

附件1（5905行 × 42列）中存在严重的缺失值问题：

| 缺失程度 | 涉及指标 | 缺失率 |
|---------|---------|--------|
| **极高** | 乙肝系列（HBsAg, HBsAb, HBeAg, HBeAb, HBcAb） | **76%** |
| **中高** | 肝功系列（ALT, AST, GGT, TP, ALB等）、肾功（BUN, Cr, UA）、血脂（TC, TG, HDL, LDL） | **21-23%** |
| **极低** | 血常规系列（WBC, RBC, HGB, PLT等） | **0.3-0.5%** |

缺失机制判断：**MAR（随机缺失）**——同一批样本同时缺失肝功、肾功、血脂指标，可能来源于不同的体检套餐或检测批次。

---

## 二、三种插补算法详解

### 算法1：中位数插补（Median Imputation）—— 基线方法

**原理**：用每个特征列的中位数替换该列的所有缺失值。

$$x_i^{(\text{miss})} = \text{median}(x_1, x_2, \ldots, x_n)$$

**优点**：
- 计算复杂度 O(1)，极快
- 对异常值不敏感（相比均值插补）
- 不改变数据的中心位置

**缺点**：
- 严重压低方差：$\text{Var}(x_{\text{imp}}) < \text{Var}(x_{\text{true}})$
- 破坏变量间的协方差结构：$\text{Cov}(x_{\text{imp}}, y) \approx 0$
- 大量相同值导致模型无法学习缺失模式中的信息

**在本项目中的问题**：对76%缺失的乙肝指标做中位数填充（0.05），意味着76%的样本获得相同值，模型无法区分"乙肝正常"和"乙肝未检测"。

---

### 算法2：KNN插补（K-Nearest Neighbors Imputation）

**原理**：
对每个含缺失值的样本 $i$，在特征空间中找到 $K$ 个最相似的完整样本（欧氏距离），用它们的加权均值填充缺失值。

$$x_{ij}^{(\text{miss})} = \frac{1}{K} \sum_{k \in \mathcal{N}_K(i)} w_{ik} \cdot x_{kj}^{(\text{obs})}$$

其中 $\mathcal{N}_K(i)$ 是样本 $i$ 的 $K$ 个最近邻，$w_{ik}$ 是权重（通常为距离的倒数）。

**距离度量**：
$$d(i, j) = \sqrt{\sum_{m \in \text{observed}} (x_{im} - x_{jm})^2}$$

仅在两个样本都观测到的特征上计算距离。

**关键参数**：
- `n_neighbors`（K值）：默认5。K越小，局部结构保持越好但噪声敏感；K越大越平滑但可能丢失局部模式。
- `weights='distance'`：距离加权，近邻贡献更大。

**优缺点**：

| 优点 | 缺点 |
|------|------|
| 非参数方法，不做分布假设 | 对高维数据敏感（维度诅咒） |
| 保留数据的局部流形结构 | 计算复杂度 O(n²)，大数据集慢 |
| 适合医学数据的聚类结构 | K值需要调参 |
| 无需预先训练 | 缺失率>50%时效果急剧下降 |

**在糖尿病数据中的适用性**：适合缺失率<20%的生化指标，但对76%缺失的乙肝指标无效（找不到足够的完整邻居）。

---

### 算法3：MICE插补（链式方程多重插补）⭐ 推荐

**全称**：Multivariate Imputation by Chained Equations

**核心思想**：
将缺失值填充转化为一系列条件回归问题。每个有缺失的特征被当作因变量 $y$，其他特征作为自变量 $X$，建立预测模型。通过迭代方式逐步逼近联合分布 $P(X_1, X_2, \ldots, X_p)$。

**算法流程（伪代码）**：

```
1. 初始化：对所有缺失值做中位数填充
2. for t = 1 to max_iter:
     for j = 1 to p:  (对每个有缺失的特征)
       a. 将特征 j 的缺失值恢复为 NaN
       b. 以特征 j 为 y, 其他特征为 X, 在完整样本上拟合:
          y_j ~ β_0 + β_1 X_1 + ... + β_{j-1} X_{j-1} + β_{j+1} X_{j+1} + ... + β_p X_p
       c. 用拟合的模型预测 y_j 的缺失值
3. 重复直到收敛（或达到 max_iter）
```

**数学基础**：
MICE假设数据来自多元分布，从条件分布中迭代采样：

$$P(X_j | X_{-j}, \theta)$$

其中 $X_{-j}$ 表示除第 $j$ 列外的所有特征，$\theta$ 是模型参数（默认贝叶斯岭回归）。

**迭代顺序**（`imputation_order='ascending'`）：
缺失率低的列先填充，这样后续列可以利用更完整的特征集做预测。

**收敛诊断**：
检查连续两次迭代的参数估计变化是否小于容差。

**优缺点**：

| 优点 | 缺点 |
|------|------|
| 能处理高缺失率（>20%） | 计算量大 |
| 保留变量间的协方差结构 | 假设数据近似正态分布 |
| 比单值插补更准确 | 需确保没有完美多重共线性 |
| 反映缺失值的不确定性 | -- |

**为什么MICE最适合本项目**：
1. 生化指标缺失21-23%：MICE可通过肝功-肾功-血脂之间的医学相关性进行合理预测
2. 乙肝指标缺失76%：MICE通过其他指标（年龄、性别、肝功）间接预测
3. 多重共线性处理：MICE内部的贝叶斯回归对共线性有一定的鲁棒性

## 三、三种算法的对比总结

| 维度 | 中位数插补 | KNN插补 | MICE插补 |
|------|-----------|---------|---------|
| 算法类别 | 单值统计插补 | 基于距离的插补 | 基于模型的迭代插补 |
| 复杂性 | O(1) | O(n²·p) | O(iter·n·p²) |
| 缺失率<5% | ✓ 最优 | ✓ 良好 | △ 过度 |
| 缺失率5-20% | ✗ 不可靠 | ✓ 推荐 | ✓ 良好 |
| 缺失率>20% | ✗ 不可靠 | ✗ 邻居不足 | ✓ **最优** |
| 保留协方差 | ✗ | ✓ 局部 | ✓ 全局 |
| 对极端值敏感度 | 低 | 高 | 中 |
| sklearn实现 | `fillna(median)` | `KNNImputer` | `IterativeImputer` |

## 四、对本项目的实践建议

### 分层处理策略

基于上述分析，建议对三类缺失采取不同方法：

```
乙肝指标（76%缺失）
  ├── 保留原始值（中位数填充）
  ├── 添加缺失指示变量（HBsAg_missing）
  └── 不参与MICE迭代建模（missing_threshold=0.5）

生化/血脂/肾功（21-23%缺失）
  └── MICE迭代建模，利用特征间相关性预测

血常规指标（0.3-0.5%缺失）
  └── 简单中位数填充即可
```

### 与现有研究对照

1. **IDF《全球糖尿病地图》** 指出全球约50%的糖尿病患者未被确诊——这印证了缺失数据本身可能携带信息。
2. **中国慢性病前瞻性研究（CKB, Lancet 2017）** 发现肝酶（ALT, GGT）与糖尿病风险显著相关——MICE通过保留肝功-血糖的协方差结构，优于中位数插补。
3. **新加坡糖尿病队列研究（Diab Care, 2020）** 使用多重插补处理缺失值，发现TG/HDL比值是糖尿病预测的重要指标。
