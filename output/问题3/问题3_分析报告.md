## 第1步：数据加载与目标定义

### 代码分析（`load_data` 函数，t3.py 第386-394行）

```python
def load_data(dm_threshold=6.1):
    df = load_and_rename(CSV1)         # 从 CSV1 加载原始数据
    df = impute_median(df)             # 中位数填补缺失值

```

- 从 `附件1：有血糖值的检测数据.csv` 读取5,905条记录
- 中位数填补（稳健处理，不受极端值影响）
- 糖尿病定义：**血糖 > 7.0 mmol/L**（WHO标准，对应空腹血糖诊断阈值）
- 生成二分类目标变量 `DM`（0=非糖尿病, 1=糖尿病）

### 终端输出对应

```
样本: 5905, DM阳性率: 8.04% (阈值>7.0)
```

**分析：** 8.04%的阳性率（475/5905）是典型的"稀有事件"分类问题，这对任何模型都是挑战。不平衡比为约11.4:1（阴性:阳性）。

### 数据分布含义
- 数据集高度不平衡 → PPV天然偏低（即使高质量模型也会有很多假阳性）
- 应优先关注 N**PV（阴性预测值）** 和 **灵敏度** 而非准确率
- F2-score（加重召回率）比 F1 更适合评估

**设计要点：**
- `StratifiedKFold` 确保每折的阳性率与总体一致（8.04%）
- 每折内 GAM 和 XGBoost 从零开始独立训练 → 无任何信息跨折泄漏
- 两个模型的概率简单平均后作为集成预测
- 每个样本的 OOF（Out-of-Fold）预测来自一个从未见过它的模型组合

### 终端输出 — 逐折性能

```
--- Fold 1/5 ---
GAM=0.7994  XGB=0.7732  集成=0.7939

--- Fold 2/5 ---
GAM=0.8061  XGB=0.7684  集成=0.7970

--- Fold 3/5 ---
GAM=0.8402  XGB=0.7761  集成=0.8216

--- Fold 4/5 ---
GAM=0.8200  XGB=0.7780  集成=0.8091

--- Fold 5/5 ---
GAM=0.8036  XGB=0.7827  集成=0.8022
```

### 逐折深度分析

| 折 | GAM AUC | XGB AUC | 集成AUC | 分析与解释 |
|---|---------|---------|---------|-----------|
| 1 | 0.7994 | 0.7732 | 0.7939 | 集成略低于GAM，XGB表现较弱拉低均值 |
| 2 | 0.8061 | 0.7684 | 0.7970 | GAM稳定>0.8，XGB持续落后~0.04 |
| 3 | **0.8402** | 0.7761 | **0.8216** | **GAM最强折**，训练/测试分割可能恰好更有区分度 |
| 4 | 0.8200 | 0.7780 | 0.8091 | GAM维持高位，XGB始终~0.77-0.78 |
| 5 | 0.8036 | 0.7827 | 0.8022 | 集成与GAM几乎相等 |

**关键观察：**
1. **GAM全面优于XGBoost**：在所有5折中，GAM 的 AUC 高于 XGBoost 的 AUC。差距在 0.022–0.064 之间。这强烈提示数据中的信号具有**平滑非线性**结构，适合 GAM 的样条基函数，而非树模型的分段常数逼近。
2. **XGBoost 性能稳定但偏低**：区间 [0.768, 0.783]，标准差仅 0.0047，说明 XGBoost 对数据分割不敏感但能力有限。
3. **集成在多数折介于两者之间**：简单平均在 4/5 折介于 GAM 和 XGB 之间。但在第3折，XGB 拖累了 GAM（0.840→0.822），说明当 GAM 表现特别好时，简单平均无法保留优势。
4. **改进建议**：如果使用 Stacking（第二层 meta-learner 学习 GAM 和 XGB 的加权组合），可能获得优于单独 GAM 的结果。

### OOF整体结果

```
=== OOF整体: AUC=0.8040, PR-AUC=0.2368 ===
```

**为何 OOF AUC(0.804) 低于 Fold 3 GAM(0.840)？**
- OOF 是**累计所有折的预测**，覆盖全部 5,905 个样本
- 第3折 GAM 的 0.840 仅在该折的 ~1,181 个测试样本上评估
- 随机分割导致了该折的测试集恰好"更易区分"，不代表模型整体能力
- OOF 的 0.804 是对 5,905 个样本**平均难度**的诚实估计

## 第7步：五档风险分层分析

### 代码分析（`evaluate_stratified`，t3.py 第204-260行）

**分层方法：**
- 基于预测概率的分位数：P0, P10, P30, P50, P80, P100
- 5档：极低 [0, P10) → 低 [P10, P30) → 中 [P30, P50) → 高 [P50, P80) → 极高 [P80, 1]
- 每档独立计算阳性率 + 累积指标

### 终端输出

```
  极低风险      n=  591 (10.0%)  阳性率=0.34%
  低风险        n= 1181 (20.0%)  阳性率=0.68%
  中等风险      n= 1181 (20.0%)  阳性率=2.79%
  高风险        n= 1771 (30.0%)  阳性率=9.32%
  极高风险      n= 1181 (20.0%)  阳性率=22.61%
```

### 分层梯度验证

```
0.34% → 0.68% → 2.79% → 9.32% → 22.61%
       2.0x      4.1x      3.3x      2.4x
```

**每一档阳性率都高于前一档，且呈加速增长。** 从"极低风险"到"极高风险"，阳性率富集了 **66.5倍**（0.34% → 22.61%）。

### 临床决策映射

| 风险等级 | 占比 | 阳性率 | 与基线(8.04%)比较 | 临床建议 |
|---------|------|--------|------------------|---------|
| 极低风险 | 10% | 0.34% | 1/24 基线 | **可安全排除**，NPV>99.4% |
| 低风险 | 20% | 0.68% | 1/12 基线 | 常规随访，无需特殊检查 |
| 中等风险 | 20% | 2.79% | 1/3 基线 | **灰区**，建议生活方式干预+OGTT |
| 高风险 | 30% | 9.32% | 略高于基线 | 建议糖化血红蛋白或OGTT |
| 极高风险 | 20% | 22.61% | **2.8x 基线** | 强烈建议临床确诊 |

```
高风险+极高风险合并为阳性: 灵敏度=0.909, 特异度=0.536, PPV=0.146
```

**灵敏度 90.9%** 意味着 475 个糖尿病患者中的约 432 个被正确识别。这是筛查场景下可接受的灵敏度的参考值。

**预期视觉形态：**
- 曲线从左上角（高精确率）迅速向右下角下降
- 精确率在召回率>0.6 后急剧下降

**判读：**
- PR-AUC=0.2368 在**极度不平衡数据**（8%阳性率）下是合理的
- 随机分类器的 PR-AUC = 阳性率 = 0.0804
- 模型 PR-AUC=0.2368 是随机分类器的 ~2.95 倍 → 模型有真实区分能力
- 曲线显示：当召回率 < 0.4 时，精确率可保持在 0.25-0.3；但当召回率 > 0.8 时，精确率下降到 ~0.18
- **临床权衡**：如果要检出 80% 的糖尿病患者，PPV 仅约 18%（每 100 个阳性预测中有 18 个真阳性）

**为什么选择 F2 而不是 F1：**
- F1 在 PR 曲线上最大化时对应的阈值通常较高（精确率优先）
- F2（beta=2）将召回率权重设为精确率的 2 倍，更适合筛查场景
- 从数据看，F2最优阈值(0.4627) 低于 Youden 阈值(0.4338)的特异度但获得了更高的 F2 分数

### 图片8：偏效应图（问题3_偏效应图.png）

**生成代码：** `plot_partial_logistic`（t3.py 第719-788行）

```
3列 × 5行（共13个特征子图）
红色实线：偏效应（log(OR)尺度）
红色阴影：95% Bootstrap置信区间
灰色虚线：y=0（参考线）
女性=0, 男性=1（性别子图）
```

**每个子图的解读方法：**

每个偏效应子图的纵轴是 **log(OR)**（对数胜算比），表示该特征取值相对于所有特征均值的风险差异：
- log(OR) > 0 → 增加糖尿病风险
- log(OR) < 0 → 降低糖尿病风险
- 阴影越宽 → Bootstrap 对该区间的不确定性越大

**逐特征预期形态分析：**

| 特征 | 预期形态 | 临床解释 |
|------|---------|---------|
| **年龄** | 从左到右缓慢上升，>60岁加速 | 年龄是糖尿病最重要的非修饰风险因素 |
| **甘油三酯(TG)** | 在低值平坦，>2.0mmol/L后快速上升 | 高 TG 是胰岛素抵抗的核心标志 |
| **红细胞计数(RBC)** | 可能呈 U 型 | 过低（贫血）和过高（血液粘稠）都增加风险 |
| **MCHC** | 中位值附近风险最低 | 反映血红蛋白浓度与血糖的复杂关系 |
| **血红蛋白(HGB)** | 随 HGB 升高而上升 | 高 HGB 可能反映糖化血红蛋白升高 |
| **ALT** | 在正常范围内平坦，>40U/L后上升 | 肝酶升高提示非酒精性脂肪肝 |
| **性别** | 两个离散点：男(1) vs 女(0) | 男>女（在 log(OR) 尺度上性别差异） |
| **LDL-C** | 随 LDL-C 升高而上升 | "坏胆固醇"的剂量效应 |
| **红细胞压积(HCT)** | 类似 HGB 形态 | 血液流变学指标 |
| **白细胞计数(WBC)** | 从低到高缓慢上升 | 慢性炎症的剂量效应 |
| **总胆固醇(TC)** | 可能呈倒 U 型 | 极度低胆固醇也可能反映营养不良 |
| **血小板计数(PLT)** | 平坦或轻微上升 | 炎症/凝血相关的间接信号 |
| **年龄_TG** | 3D交互效应投影 | 年龄增大强化了 TG 的风险效应 |

**置信区间解读：**
- 在数据稀疏的区域（如极高 TG 值），Bootstrap 置信区间会变宽
- 在数据密集的区域（如 年龄 40-60岁），置信区间较窄
- 如果某个特征的置信区间在大部分取值范围内包含 y=0 → 该特征的独立风险贡献可能不显著

## 第9步：模型对比

### 代码分析（`compare_models`，t3.py 第900-965行）

- 在**全量数据**上做 5 折 StratifiedKFold
- 每折独立训练 GAM / RF / XGBoost
- 计算每折的 AUC 和 PR-AUC

### 终端输出

```
模型                   AUC (5-CV)   PR-AUC (5-CV)
Logistic GAM         0.8138       0.2723
RandomForest         0.7864       0.2187
XGBoost(cal)         0.7780       0.2120
```

### 分析

| 模型 | AUC | vs GAM 差距 | PR-AUC | vs GAM 差距 |
|------|-----|------------|--------|------------|
| **Logistic GAM** | **0.8138** | — | **0.2723** | — |
| RandomForest | 0.7864 | -0.0274 (-3.4%) | 0.2187 | -0.0536 (-19.7%) |
| XGBoost(cal) | 0.7780 | -0.0358 (-4.4%) | 0.2120 | -0.0603 (-22.1%) |

**为什么 GAM 明显优于树模型？**

1. **信号-噪声比低**：血检指标与糖尿病风险的关系本质上是平滑的、单调的或简单的非线性关系，不需要复杂的高阶交互。GAM 的样条基函数恰好适合这种信号结构。

2. **树模型在此场景的劣势**：
   - RandomForest/XGBoost 依赖递归分区 → 对"弱信号"数据的利用率较低
   - 8% 阳性率下树模型的分类边界可能不稳定
   - XGBoost 虽然通过 CalibratedClassifierCV 校准，但基学习器仍然是树

3. **为什么 XGBoost 比 RF 还差一点**：
   - XGBoost 的默认正则化（reg_alpha, reg_lambda）可能在此数据上过强
   - 300 个树可能不够（对稀有事件通常需要更多树）
   - 经过 Platt Scaling 校准后，概率分布被压缩，PR-AUC 受到一定影响

4. **GAM 的额外优势**：
   - 可解释性：偏效应图可直接展示每个特征的风险贡献形态
   - 样本权重自然支持：`compute_class_weight('balanced')` 直接传入 fit()
   - 校准性：Logit 链接本身输出校准概率
